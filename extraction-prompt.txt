# Basketball Ireland / domestic scoresheet layout (`extraction-prompt.txt`)
# Pair with `schema.json`. For FIBA international sheets use `extraction-prompt-fiba.txt` + `schema-fiba.json` (select “FIBA / international” on upload).

---

Document layout (read this first):
- The game box is split into **left (L)** and **right (R)**. The **header** spans the full width above L and R (not split into two sides).
- **L** has three sections top to bottom: **L1** (Team A / home), **L2** (Team B / away), **L3** (officials).
- **R** has three sections: **R1** (running score), **R2** (period scores), **R3** (final).

**FIBA international scoresheet** (same JSON schema): Header may show **Competition**, **Game No.** → **gameNumber**, Date, Time, Place; **Referee** → **referee1**; **Umpire 1** → **referee2**; **Umpire 2** → **umpire2Name**. Coach rows use **Licence no.** → **teamACoachBipin** / **teamBCoachBipin**. Officials block may list Scorekeeper, **Assistant Scorekeeper** → **assistantScorekeeperName**, Timekeeper, **24" operator** (shot clock). **Scores** section: periods labelled j,k,l,m (four quarters) plus a separate **Extra periods** row → **r2ExtraPeriodPointsTeamA** / **r2ExtraPeriodPointsTeamB** (points in OT only; 0 if no overtime). Running score may use **four** 40-point blocks (plays **1–160**); domestic sheets often use three blocks (**1–120**).

Header (three rows, full width):
- Row 1: Team A name, Team B name.
- Row 2: Competition name, date, time (plus **Game No.** on FIBA → **gameNumber**).
- Row 3: Place/venue, referee 1, referee 2 (when shown). FIBA: Referee + Umpire 1 + Umpire 2 as above.
- Map to JSON: teamAName, teamBName, competitionName, date (YYYY-MM-DD), time (HH:MM 24h), place, referee1, referee2, and when present gameNumber, umpire2Name.

L1 – Team A (home):
- **Team A header block** (top of L1): team name after “Team A:”, **team colour** after “Colour” (extract to **teamAColour**). Below: **time-outs** (first half, second half, extra periods) and **team fouls** (one row of boxes per period, four boxes per period).
- **Time-outs / team fouls notation:** A slash (/) or backslash (\) in a box = one time-out or one team foul used in that slot. A **horizontal line (– or -)** through a box = not used in that period (still “filled” as unused). **Blank** = not used. Extra-period boxes with **no** slashes or lines and no extras imply **no overtime** (use 0 for extra-period time-outs). First half = 2 boxes; second half = 3; extra = 3 per extra period.
- **Team fouls:** Count **only** / or \ as a team foul. **Do not** count dashes or blanks. Integers go to teamAFoulsPeriod1..4 (0–4 each).
- **Player table:** Columns left to right: BIPIN, Player In (4 boxes Q1–Q4), Player Name, **Kit No.** (jersey), Fouls (5 boxes). One JSON object per **printed roster row** in **teamAPlayers**  -  not a sample, not “starters only,” not the first two only.
- **Roster completeness (critical):** FIBA / Oceania sheets usually have **12 horizontal player rows** in each team block (often labelled Licence no. 1–12 or similar). You must emit **one object per row from the top of that grid down to the last player row**, in order, **before** the coach / assistant coach lines. Rows with no “Player in” mark, or with empty foul boxes, **still count**  -  include **playerName** and **kitNo** whenever printed. **Do not** stop after two players. **Do not** put coach or assistant coach names in **teamAPlayers** / **teamBPlayers** (those belong in **teamACoachName**, **teamAAssistantCoachName**, etc.).
- **Player In:** X, /, or \ = played that quarter; empty = did not play that quarter. If **all four** are empty, the scorekeeper may not have filled the column (not “did not play”). Starters-only marking is common.
- **Kit No.:** Always copy into **kitNo** when visible (e.g. "07" or "7" as printed).
- **Player fouls:** P, P1, P2, P3, U, D, T; horizontal line = no foul by end of game. Fifth foul = fouled out.
- **Staff (below players):** Coach name + BIPIN; Assistant coach name (no BIPIN on some sheets). Map to **teamACoachName**, **teamACoachBipin**, **teamAAssistantCoachName** when visible.

L2 – Team B:
- **Same structure as L1** for the away team. Use **teamB** fields, **teamBPlayers** (again: **every** roster row, typically **12** on FIBA), **teamBCoachName**, **teamBCoachBipin**, **teamBAssistantCoachName**, **teamBColour**, and team B time-out / team foul integers.

L3 – Officials (below L2):
- Rows for: referee 1, referee 2, scorekeeper, **assistant scorekeeper** (FIBA), time keeper, 24-second operator; plus captain’s signature in case of protest (often **no BIPIN** for signature line).
- Emit **l3Officials** as an array of objects: { "role": "referee1" | "referee2" | "scorekeeper" | "assistantScorekeeper" | "timekeeper" | "shotClock24s" | "captainSignature", "bipinNo": "", "name": "", "initials": "" }. Use **empty string** for bipinNo when the row has no BIPIN column. Omit roles not present on the sheet.

R1 – Running score:
- **L1 = Team A only; L2 = Team B only** for rosters. In the **R1 grid**, the **left pair** of columns is Team A (jersey + point mark); the **right pair** is Team B. Do **not** swap teams.
- **Three or four** main columns of **40 plays** each (e.g. 1–40, 41–80, 81–120, **121–160** on FIBA). Each main column has **four inner columns**: (1) Team A jersey, (2) Team A point mark, (3) Team B point mark, (4) Team B jersey.
- **Point marks:** dot/small mark = 1 pt (FT); slash or backslash through the play number = 2 pts; **circle** around the play number = 3 pts. Classify only marks in the **R1 point columns**, not slashes in player “Player In” rows.
- **runningScoreEvents:** one object per **scoring play** in **chronological order**: { "point": 1..n (up to **160** on FIBA four-block sheets, **120** on three-block sheets), "team": "A" or "B", "type": "1"|"2"|"3", "jersey": "..." }. **point** is the **order of scores** (1st basket of the game, 2nd, …), not the printed row label in every layout. **jersey** is the scorer’s kit number on that row for that team.
- Ignore cells **scribbled out** vertically after the game ended; do not count those as scores.
- **pointsPerColumn:** 40 or 60 per main block depending on the sheet (40 or 60).

R2 – Period scores:
- Points scored **in each period only** (not running totals). **r2PeriodScoresTeamA** and **r2PeriodScoresTeamB**: four numbers [Q1,Q2,Q3,Q4] (FIBA may label periods j,k,l,m  -  still four quarter scores).
- If the sheet has a separate **Extra periods** row in the Scores table, fill **r2ExtraPeriodPointsTeamA** and **r2ExtraPeriodPointsTeamB** with OT points only; otherwise use **0**.

R3 – Final:
- Total final scores including overtime: **r3FinalScoreTeamA**, **r3FinalScoreTeamB**, and **r3WinningTeamName** as printed.

Also fill **periodScoresTeamA/B** and **finalScoreTeamA/B** from the main scores / final when visible (see schema).

Accuracy before returning JSON:
1. Read printed finals; set finalScoreTeamA/B and r3 fields consistently.
2. Sum of `type` (1+2+3) over runningScoreEvents per team should match that team’s final if every basket is captured and classified.
3. Do not leave kitNo blank when the Kit column is readable.
4. **Count roster rows:** **teamAPlayers.length** should match the number of player rows visible in **L1** (up to 12 on standard sheets). **teamBPlayers.length** should match **L2**. If you only extracted two players per team, re-read the full left column tables  -  the grid continues below the first two names.
5. Prefer emitting **teamAPlayers** and **teamBPlayers** in your JSON **before** **runningScoreEvents** so long play-by-play lists do not crowd out roster data.

Return a single JSON object matching the schema. Use "" for missing strings and [] for missing arrays where allowed.
